## [0.5.118] - 2026-04-25 - 记忆包写入能力与提示词修复

### Changed
- `memory-pack` 现在会挂载长期记忆和敏感长期记忆的写入、更新、删除、建议与确认工具，不再只暴露读取和预览工具。

## [0.5.111] - 2026-04-24 - 1052-PD 首版落地

### Added
- 新增 `1052.md` 作为 `1052-PD` 的核心规则入口，并新增 `agent.pack / checkpoint / seed / p0 / redaction / upgrade / runtime types` 等后端模块，支撑单核渐进披露、checkpoint 续跑和老会话 seed。
- 新增只读终端工具 `terminal_run_readonly`，以及 `agent.progressive.test.ts` 回归测试，覆盖 pack 归并、升级上限与 checkpoint 注入预算。

### Changed
- 后端 `agent.service` 默认切换到 `1052-PD` 渐进披露链路：`P0` 仅保留 `request_context_upgrade` 元工具，按需挂载 `base-read-pack / repo-pack / search-pack` 等工具包，不再首轮全量注入业务 tools schema。
- `llm.client` 补充请求中止透传、MiniMax 单 `system` 兼容保持、cache usage 解析与更完整的 token usage 归并。
- `settings`、`agent.types`、`agent.routes`、`agent.history.service`、`agent.stats.service`、前端 `api/settings` 与 `api/agent` 扩展了 `1052-PD` 开关、cache usage 和 upgrade overhead 字段。
- 设置页保存链路已支持 `progressiveDisclosureEnabled / providerCachingEnabled / checkpointEnabled / seedOnResumeEnabled / upgradeDebugEventsEnabled` 五个新开关。

### Fixed
- SSE 聊天流在客户端中止或连接关闭时，后端现在会同步中止底层 LLM 请求，避免旧请求继续占用模型和工具轮次。
- UAPIs 运行时上下文新增轻量目录摘要接口，避免在渐进披露链路中重新注入完整 UAPIs 大索引。
- 修复 Windows 下 checkpoint `sessionId` 使用 `:` 导致的异常文件路径问题；`web:default` 等旧格式现在会归一化为 `web-default`，并支持从旧路径兜底迁移读取。
